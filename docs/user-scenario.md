#  Користувацькі сценарії SkillSwap AI

## Сценарій : Перший матчинг та урок 
### Передумови
- Користувач має акаунт з профілем (навички, часові пояси, цілі)
- На платформі мінімум 2 користувачі з доповняльними навичками

### Кроки сценарію

1. **Користувач заходить на сторінку "Пошук партнера"**
   - Бачить запрошення: "Що хочеш навчити? Хто тебе навчатиме?"
   - Вибирає навичку для навчання (наприклад, "Python для початківців")
   - Вибирає навичку для обміну (наприклад, "Гітара для новачків")

2. **ШІ аналізує й пропонує партнерів**
   - Виводиться карточка партнера:
     - Фото, ім'я, рівень скіл-сету (зірки), кількість успішних уроків
     - Геолокація / часовий пояс
     - Коротка біографія ("Python-розробник 3 роки, люблю вчити новачків")
     - Оцінка від інших учнів (якщо є) — "4.8/5 від 12 учнів"
   - Кнопка "Запросити урок" та "Далі" (переглянути іншого)

3. **Користувач обирає партнера**
   - Натискає "Запросити урок"
   - З'являється форма для бронювання:
     - Вибір часу (календар з доступними слотами партнера)
     - Тривалість уроку (30/60/90 хвилин)
     - Короткий опис того, що хочеш вивчити (текстове поле)
   - Натискає "Відправити запит"

4. **Партнер отримує сповіщення**
   - Push/Email: "Нова пропозиція уроку від [ім'я]"
   - Переглядає карточку запрошувача (рівень скіл-сету, їхні очікування)
   - Натискає "Прийняти" або "Відхилити"

5. **При прийнятті — підтвердження**
   - Обом учасникам приходить підтвердження:
     - Дата, час, тривалість
     - Посилання на відеочат (відкривається прямо в додатку)
     - Чат для спілкування до уроку

6. **У день уроку**
   - За 15 хвилин — нагадування обом
   - За 5 хвилин — можливість вайтрумути й перевірити мікрофон/камеру
   - По закінченню — система пропонує залишити оцінку й коментар

### Очікуваний результат
 Два користувачі з успішно проведеним уроком
 Обидва залишили оцінку 4.0+ з 5
 Обидва отримали "часові токени" за участь
 Система записала цю успішну пару для аналізу (щоб вдосконалювати алгоритм)

### Альтернативні потоки

**A1: Партнер відхилив запит**
- Запрошувачу пропонується наступна карточка
- Можливість записати в список очікування ("Повідомте мене, коли партнер буде вільний")

**A2: Користувач скасував урок**
- До 24 годин до урока — без штрафу
- Менше 24 годин — втрата 50% часових токенів (на майбутньому розширенні)

**A3: Технічна проблема з відеочатом**
- Система автоматично запускає резервну платформу (або давидений телефонний зв'язок)
- Обидва учасники могли переносити урок без штрафу

## Сценарій : Верифікація рівня навичок

### Передумови
- Новий користувач створив профіль і позначив себе експертом у Python

### Кроки сценарію

1. **Система запропонувала верифікацію**
   - Спливаюче вікно: "Підтвердіть ваш рівень Python для більшої надійності"
   - 3 варіанти:
     - Пройти тест (5 запитань базового рівня)
     - Завантажити сертифікат (coursera, udemy)
     - Чекати відгуків від учнів (зберуть автоматично після 3+ уроків)

2. **Користувач пройшов тест**
   - Правильні відповіді на 4+ з 5 → отримує значок ✓ "Python — верифіковано"
   - На профілі з'являється бейдж: "Верифіков. експерт" (підвищує довіру)

3. **Як це вплинуло**
   - Пропозиції розміщуються вище в пошуку
   - Новички частіше обирають верифікованих учителів

### Очікуваний результат
 Профіль надійніший
 Більше вхідних запитів
 Більша імовірність відповідати запитам

## Сценарій : Управління часом та часовими поясами

### Передумови
- Учитель у Португалії (GMT+0), учень у Японії (GMT+9)
- Обидва мають обмежений час (8-10 ранку за своїм часом)

### Кроки сценарію

1. **Учитель встановлює свої доступні слоти**
   - Переходить в "Мій календар"
   - Вказує: "Доступний пн-пт 8-10 ранку (Лісабон)"
   - Система АВТОМАТИЧНО перетворює на 5-7 вечора (Токіо)

2. **Учень пошукує партнерів для навчання**
   - Введе фільтр: "Python для новачків"
   - Система пропонує партнерів, у яких є оверлап часу
   - Карточка покажеться, тільки якщо є МОЖЛИВІ слоти обох

3. **При бронюванні**
   - Календар показує час обох часових поясів:
     - "Пн, 10:00 (Лісабон) / Пн, 19:00 (Токіо)"
   - Можна змінити налаштування мови для часів

### Очікуваний результат
 Без плутанини з часовими поясами
 Реальний оверлап часу гарантований
 Обидва бачать зручний для себе час

## Сценарій : Система репутації та часових токенів

### Передумови
- Користувач провів 3 успішних уроки (9 годин навчання отримав, 9 годин дав)

### Кроки сценарію

1. **Користувач переглядає свій профіль**
   - Бачить рахунок часових токенів:
     - Заробив: 8 годин 45 хвилин
     - Витратив: 9 годин 15 хвилин
     - Баланс: -30 хвилин (борг, але це норм для новачків)
   - Історія уроків із оцінками

2. **Система підраховує репутацію**
   - **Оцінки учнів** (40%): Середня 4.7/5 → Бал 94/100
   - **Пунктуальність** (30%): 3 з 3 прийшли вчасно → 100
   - **Отримані відгуки** (30%): "Чудовий учитель", "Енергійний", "Дякую!" → 90
   - **Загальна репутація**: 94/100 → Значок "Надійний партнер" 
3. **Ефект для користувача**
   - Пропозиції матчингу приходять частіше
   - Запити приймаються швидше (люди довіряють)
   - Можна відфільтрувати шукачів за мінімальною репутацією

### Очікуваний результат
 Чесна система, що заохочує якість
 Новачки можуть зростати в репутації
 Платформа самоочищується від недобросовісних осіб


##Критерії приймання:
Реєстрація та заповнення профілю займає не більше 3 хвилин.
Алгоритм ШІ видає мінімум 3 релевантні пропозиції для матчингу.
Кнопка виклику відеозв'язку стає активною одразу після взаємної згоди сторін.
Система автоматично фіксує тривалість уроку.

###План перевірки 
Гіпотеза: Користувачі готові витрачати свій час на навчання інших без грошової винагороди.
Метод перевірки: Проведення 50 тестових сесій між реальними користувачами протягом тижня.
Метрика успіху: Більше 60% учасників бета-тесту погоджуються на повторну сесію з тим самим або іншим партнером.

###Ризики 
Ризик: Велика різниця у рівнях знань партнерів (наприклад, профі та новачок), що зробить обмін неефективним.
Рішення: Додати в анкету самооцінку рівня володіння навичкою (Junior, Middle, Senior) для точнішогодо матчингу.
